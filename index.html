Описание архитектуры системы

Информационная система гимназии

Использование искусственного интеллекта для выполнения административных и учебных задач в гимназии им. В.В. Давыдова.

Раздел I. Принципы внедрения

Автоматизация административной работы

Система развивающего обучения Д.Б. Эльконина — В.В. Давыдова требует от учеников самостоятельного поиска решений. Стандартные программы с искусственным интеллектом (ИИ) сразу выдают готовые ответы. По этой причине они не подходят для данной учебной методики в неизменном виде.

Основная функция внедряемой системы — выполнение административной работы. Программы собирают данные, сверяют расписание и подготавливают шаблоны документов. В результате система берет на себя рутинные действия и снижает нагрузку на учителей и руководство гимназии.

Разделение задач между программами

Работа системы строится на основе мультиагентной архитектуры (подход, при котором общая задача делится между несколькими отдельными программами, как распределение обязанностей между узкопрофильными специалистами). Каждая программа-агент выполняет свою функцию:

  • Программа-скаут: регулярно проверяет государственные сайты и находит информацию о финансировании или грантах.
  • Программа-аналитик: сопоставляет данные школьного расписания и оценок из электронных журналов для поиска накладок.
  • Программа-стратег: объединяет данные от других программ и формирует черновики отчетов или заявок.

Главная программа (Оркестратор) распределяет задачи между агентами и сохраняет историю всех действий в общую базу данных. В школьной системе такой подход обеспечивает доступ к функциям даже при сбое одной отдельной программы.

Раздел II. Организация данных

Образовательный цифровой двойник

Основой работы системы выступает цифровой двойник (виртуальная копия гимназии, объединяющая профили учеников, преподавателей и учебных материалов в единую сеть, подобно профилям и контактам в социальной сети).

Двойник связывает списки классов, расписание звонков и оценки в общую структуру. Это позволяет программам анализировать информацию комплексно, а не в отдельных файлах.

Анализ связей и расписания

Программы постоянно анализируют информацию из цифрового двойника. Например, изменение расписания в календаре одного преподавателя автоматически пересчитывается для других классов. Это позволяет руководству заранее отследить перегрузку сотрудников или ошибки в графике уроков.

Автоматизация поиска финансирования

Программа-скаут сравнивает профиль гимназии (число учеников, направления) с условиями новых государственных программ на сайтах заявок. При совпадении критериев программа-стратег самостоятельно составляет предварительный текст документа. В результате администрация получает готовый черновик для утверждения.

Раздел III. Безопасность и хранение данных

Поэтапный запуск системы

Законодательство (закон о персональных данных ФЗ-152) строго регулирует хранение и обработку школьной информации. Обработка реальных данных учеников на зарубежных серверах в интернете юридически запрещена.

Для соблюдения законодательства внедрение системы разделено на три этапа. Это позволяет настроить программы без риска утечки информации.

Этапы настройки программ

Этап 1: Тестирование

Использование внешних открытых сервисов (подобных обычным онлайн-переводчикам) только на тестовых, вымышленных данных. Цель этапа — проверка общих связей в системе и подготовка сотрудников без использования реальных документов гимназии.

Этап 2: Переходный период

Разделение информационных потоков. Открытые данные обрабатываются через внешние сервисы. Конфиденциальная информация об учениках остается только во внутренней сети школы.

Этап 3: Полная автономность

Установка всех программ ИИ на серверы (мощные системные блоки) внутри здания школы. Такое локальное размещение обеспечивает полный контроль над хранением информации и соответствие требованиям законодательства РФ. Доступ к интернету для обработки данных больше не требуется.

Раздел IV. Логика работы с учениками

Организация учебного процесса

Стандартные программы искусственного интеллекта работают как поисковики: они сразу пишут готовый текст решения по запросу пользователя. В условиях гимназии механизм программ специально изменен.

Алгоритм настраивается на работу по принципу сократического диалога (метод ведения беседы, при котором программа не дает готовый ответ, а задает наводящие вопросы). Это необходимо для соответствия нормам развивающего обучения.

Взаимодействие ученика и системы

При обращении ученика с задачей программа действует по заранее заданному сценарию. Каждое действие выполняется последовательно:

Шаг 1. Проверка ответа

Если ученик дает неполный ответ, программа задает уточняющий вопрос. Это выявляет слабые места в рассуждениях школьника.

Шаг 2. Добавление противоречий

Алгоритм сообщает новые факты, которые не вписываются в текущую логику ученика. Возникает потребность в пересмотре выводов.

Шаг 3. Использование подсказок

Программа задает серию мягких наводящих вопросов.

Шаг 4. Формирование окончательного вывода

Алгоритм объединяет ответы ученика и помогает ему самостоятельно сформулировать правильное решение правила или задачи. В результате такого взаимодействия самостоятельность мышления школьника сохраняется.
